## 使用说明

### 启动项目

```bash
cd admin-web
npm install
npm run dev
```

### 测试账号

- **系统管理员**: `admin` / `123456`
- **题库管理员**: `content` / `123456`
- **代理商**: `agent` / `123456`

### 开发建议

1. **Mock 数据**: 当前使用简单的 Mock 数据，实际开发时需要对接真实后端接口
2. **类型定义**: 建议为所有 API 接口定义 TypeScript 类型
3. **错误处理**: 完善错误处理机制，提供友好的错误提示
4. **性能优化**: 对于大数据量列表，考虑使用虚拟滚动
5. **测试**: 添加单元测试和 E2E 测试

## 项目结构说明

```
admin-web/
├── src/
│   ├── api/              # API 接口定义
│   │   ├── user.ts       # 用户相关接口
│   │   ├── dashboard.ts  # 仪表盘接口
│   │   ├── question.ts   # 题库管理接口
│   │   ├── agent.ts      # 代理商接口
│   │   └── system.ts     # 系统管理接口
│   ├── views/            # 页面组件
│   │   ├── dashboard/    # 仪表盘
│   │   ├── question/     # 题库管理
│   │   ├── agent/        # 代理商中心
│   │   ├── user/         # 小程序用户
│   │   ├── system/       # 系统管理
│   │   ├── login/        # 登录页
│   │   └── error/        # 错误页
│   ├── layouts/          # 布局组件
│   ├── store/            # 状态管理
│   ├── router/           # 路由配置
│   ├── utils/            # 工具函数
│   └── mock/             # Mock 数据
├── package.json
├── vite.config.ts
└── README.md
```

## 注意事项

1. **环境变量**: 需要配置 `.env` 文件，设置 API 基础地址
2. **Mock 数据**: 当前 Mock 数据较为简单，实际使用时需要完善
3. **权限控制**: 权限验证逻辑已实现，但需要后端配合返回正确的角色信息
4. **富文本编辑器**: TinyMCE 需要正确配置，特别是 LaTeX 公式插件
5. **文件上传**: 当前文件上传功能为占位实现，需要对接真实的上传接口

## 后续开发建议

1. **完善类型定义**: 为所有数据结构定义 TypeScript 接口
2. **添加单元测试**: 为核心功能添加单元测试
3. **性能优化**: 优化首屏加载速度，使用代码分割
4. **国际化**: 如需支持多语言，可集成 vue-i18n
5. **主题定制**: 支持主题切换功能
6. **日志系统**: 添加操作日志记录功能
